## 沪美IPLC是什么：一条不走公网的"私人高速公路"

先说清楚概念。IPLC是International Private Leased Circuit的缩写，翻译过来叫"国际私有专线"。关键词在"私有"两个字——这条线不跟别人共享，不过公网路由，不走GFW的检测逻辑。你从上海入口进去，从美国出口出来，中间这段路是封闭的。

沪美IPLC的具体架构是这样的：

- **入口**：上海电信（也有上海BGP和上云互联优化IXP两种入口可选）
- **出口**：美国BGP（实测落地在洛杉矶，AS51847 Nearoute Limited）
- **端内延迟**：官方标称124~134ms
- **IP配置**：双独立IPv4，一个入口IP一个出口IP

这里有个容易让人误解的地方。124ms这个数字，指的是"端内延迟"，也就是从上海入口节点到美国出口节点这段专线的延迟。不是你从家里电脑ping美国服务器的延迟。你实际使用时，还要加上"你家到上海入口"这一段的延迟。不过这一段走的是国内骨干网，国内三网到上海的延迟通常在20~35ms之间，所以总延迟大概是150~170ms的样子。

## 常见问题

**Q：124ms的延迟，网页操作会卡吗？**

不会。网页操作的延迟感知阈值通常在200ms以上，124ms加上你到上海的20~35ms，总延迟150ms左右，打开网页、点击按钮、提交表单这些操作基本无感。只有高频实时交互（比如游戏、远程桌面拖拽文件）才会明显感知。

**Q：双向流量是什么意思？**

双向流量指上传+下载的总量。100GB双向流量，意味着上传和下载加起来不能超过100GB。实际使用中，如果你以下载为主，单向可用流量大约是标称的一半。

**Q：可以换省份吗？**

可以。产品开通后可以修改白名单省份，但不能同时支持多省份。如果你有多个地区的办公需求，需要分别购买或者每次切换省份。

**Q：晚高峰会掉速吗？**
